# 大数据技术整理

> 一台机器存不下、算不动 → 用一堆机器一起干

## 采集（数据怎么来）

| 技术 | 用途 |
|------|------|
| **Kafka** | 实时消息管道，最核心 |
| **Flume** | 服务器日志采集 |
| **Canal** | 监听 MySQL 变更实时同步 |
| **DataX** | 多数据源批量搬运（阿里开源） |

## 存储（数据存哪里）

| 技术 | 适合场景 |
|------|----------|
| **HDFS** | 海量文件分布式存储 |
| **HBase** | 百亿行按 ID 秒查 |
| **ClickHouse** | 统计分析极快（列式） |
| **Elasticsearch** | 全文搜索 + 日志分析 |
| **Redis** | 内存缓存、实时计数 |
| **TDengine** | IoT 时序数据 |

## 计算（数据怎么算）

| 技术 | 类型 | 一句话 |
|------|------|--------|
| **Spark** | 批处理 | 攒一批一起算，主流 |
| **Flink** | 流处理 | 来一条算一条，实时最强 |
| **Hive** | SQL 查大数据 | 写 SQL 出报表 |
| **Presto/Trino** | 交互式查询 | 秒级返回 |

## 调度（什么时候算）

| 技术 | 说明 |
|------|------|
| **Airflow** | 全球最流行的任务编排 |
| **DolphinScheduler** | 国产可视化调度 |

## 可视化（数据给谁看）

| 技术 | 说明 |
|------|------|
| **Grafana** | 实时监控仪表盘 |
| **Superset** | 数据分析报表（Apache 开源） |
| **DataV** | 大屏可视化（阿里云） |

## 常见架构

**实时**：`Kafka → Flink → ClickHouse → Grafana`

**离线**：`DataX → HDFS → Spark + Hive → Superset`

**小团队**：`MySQL → Grafana`（够用就行）

## 核心四件套

| Kafka | Flink | Spark | ClickHouse |
|-------|-------|-------|------------|
| 数据管道 | 实时算 | 批量算 | 快速查 |
